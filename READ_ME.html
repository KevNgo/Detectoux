Architecture du Logiciel et du Projet

Récupération d’une base de données exploitable

Nous récupérons les données sur github.

Il contient 121 extraits sonores différents de toux labelisés positif/négatif au Covid19, ainsi que des informations dont les antécédents médicaux, les symptômes et les fumeurs. Ces toux sont celles de 17 personnes et cette base de données contient uniquement 17 lignes dans le tableau.

Mise en forme de la base de données

La première chose à faire a été de dupliquer les lignes pour chaque toux enregistrée jusqu’à obtenir 121 lignes contenant le nom de l’extrait audio associé.

Traitement du signal

Pour extraire des informations sonores, nous utilisons la librairie librosa.
Nous avons donc utiliser les fonctions de librosa pour récupérer de nombreuses informations issues entre autres de la fréquence et du spectre.
Ces données sont ensuite ajoutées au tableau de données.

Ajout de bruit dans nos données

Afin de doubler notre volume de données, nous avons utilisé des extraits existants auxquels nous avons ajouté du bruit et les avons traités comme de nouvelles données jusqu’à atteindre 284 lignes dans notre tableau de données.

Normalisation et exploration des données

Les paramètres catégoriques ne sont pas exploitables pour certains modèles, il est donc nécessaire de les passer en données numériques.
Nous utilisons la fonction LabelEncoder() de la librairie Scikit Learn pour les catégories binaires.
Concernant les catégories « symptômes » et « antécédents », nous avons attribuer manuellement des valeurs numériques en suivant une hiérarchie selon nos propres hypothèses.
Nous utilisons en suite les librairies matplotlib.pyplot et seaborn afin de visualiser une matrice de corrélations des paramètres. Cela nous permet de repérer les paramètres très fortement corrélés pour n’en garder qu’un par groupe et alléger les temps de traitement.

Ensuite, nous avons utilisé la fonction de mise à l’échelle MinMax de Scikit Learn pour les paramètres sonores contenant des valeurs négatives. Ceci est un prérequis pour la fonction SelectKBest de Scikit Learn qui nous permet de recherchés les paramètres les plus significatifs vis-à-vis du label « has_Covid19 ».
On ne garde alors que ceux qui obtiennent un score important dans notre tableau de données.

Recherche de modèles d’apprentissage

Nous avons suivi les recommandations de Scikit Learn en essayant dans l’ordre des modèles : SVM, KNN et Decision Tree Classifiers.
Les premiers résultats étant peu encourageants(<66%), nous avons retravaillé au fur et à mesure les données que nous utilisons pour entrainer ces modèles.

Une fois un score intéressant obtenu sur le testset avec le modèle Decision Tree Classifier (environ 85%), nous avons utilisé la validation croisée afin de tester différentes combinaisons aléatoires de traintest et testset et en sortir une précision moyenne(94%).
Toujours dans l’optique d’améliorer ce modèle, nous testons et visualisons une courbe pour déterminer sur une fourchette de [1 :30] la valeur de l’hyperparamètre « max_depth » qui donne le meilleur score. On peut voir qu’à partir de max_depth = 9, le score du trainset est maximale et celui du testset stagne autour de 93%.

Enfin, nous utilisons la fonction GridSearchCV de Scikit Learn permettant de rechercher la meilleur combinaison d’hyperparamètres du modèle qui nous retourne un score de 97% sur le trainset et un score de 96% sur le testset.
Les hyperparamètres sont réglés sur : {‘criterion’: ‘entropy’, ‘max_depth’: 9, ‘splitter’: ‘best’}
Finalement, nous utilisons une matrice de confusion qui nous retourne que les erreurs de prédictions concernent des faux positifs. Ce qui est moins contraignant qu’un faux négatif pour ce type de prédiction.

Estimation de la quantité de données suffisante

Grace à la fonction learning_curve de Scikit Learn, nous visualisons le taux d’aprentissage qui nous permet de savoir si notre modèle pourrait être améliorer en augmentant le volume de données recueilli. La courbe nous montre que nous n’avons pas atteint le maximum. Dans un contexte agile, cette base de données serait encore augmentée jusqu’à atteindre ce maximum au-delà duquel le score n’augmenterait plus.

Développement web

Lorsque l’on ouvre le site Detectoux, nous arrivons sur la page d’accueil de celui-ci. De cette page, l’utilisateur peut commencer la procédure de test en cliquant sur le bouton « Je me teste ».
La première étape est un formulaire à remplir afin d’obtenir des résultats plus précis. Une fois les informations récupérées, l’utilisateur est dirigé vers la page d’enregistrement de la toux. Sur cette page, il enregistre sa toux, peut l’écouter et l’enregistrer de nouveau si besoin. Il accède ensuite au diagnostic : il est alors dirigé vers la page de résultats négatif ou positif selon le résultat de l’analyse. Pour finir, il peut retourner sur la page d’accueil s’il souhaite se tester à nouveau ou alors tester une autre personne.
